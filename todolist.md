# FinUnity 最新产品任务板

产品原则：

```text
首页看结构
规划做决策
账户管数据
明细看变化
```

## 全局设计约束

所有任务执行时必须先遵守本节，再看具体页面任务。

### 产品气质
- 做成个人资产账本和家庭资产决策辅助工具，不做传统资产后台。
- 页面要安静、干净、轻量、柔和。
- 整体参考 iOS-like 轻量理财风格。
- 低信息密度，大留白，大圆角，弱阴影。
- 少按钮，少说明文字，少工具栏。
- 避免复杂数据大屏、后台管理表格、密集筛选、堆满功能入口。

### 视觉规范
- 页面背景统一使用浅灰：`#F7F8FA`。
- 卡片背景统一白色：`#FFFFFF`。
- 卡片圆角优先使用 `24dp`，重要大卡可用 `24dp~28dp`。
- 页面左右边距统一 `20dp`。
- 卡片内边距优先 `20dp`。
- 卡片之间间距优先 `16dp`。
- 阴影很轻或几乎没有，避免强立体感。
- 主文字颜色：`#1F2933`。
- 辅助文字颜色：`#A0A4AA`。
- 重点数字颜色：`#111827`。
- 正收益颜色：`#1E8E5A`。
- 负收益颜色：低饱和红色，避免刺眼。
- 低饱和绿色只做点缀，不要大面积铺满。
- 大金额字号突出，指标字号小。
- 页面要能同时适配 Android 和 iOS 的审美，不做 Android 后台管理风。

### 组件风格
- 首页和主流程优先使用白色大圆角卡片。
- 每个卡片只承载一个明确主题。
- 不在卡片里塞太多按钮。
- 功能入口优先做成轻卡片或低强调文字入口。
- 图标使用线性或轻量图标，避免厚重实心图标堆叠。
- 常用入口可以用大图标 + 大文字。
- 次级操作放到更多菜单、详情页或低强调入口。
- 不要把所有功能都做成并列按钮。
- 不要把 UI 做成表单后台、财务软件后台或行情交易软件。

### 信息层级
- 第一屏只回答最重要的问题。
- 总览页突出资产结构，不展示完整管理能力。
- 账户页突出账户管理，不做资产分析。
- 明细页突出变化和操作，不承载全局规划。
- 规划页突出决策建议，不做数据录入。
- 若某个模块不是当前页面的核心目标，应弱化或移到次级页面。

### 交互约束
- 底部 Tab 是根页面切换，不进入普通返回栈。
- 高频操作只保留一个主入口，例如账户详情页主操作是 `记一笔`。
- 买入、卖出、转账、收支、修正等应尽量收敛到 `记一笔` 的清晰分组里。
- 账户新增/编辑属于账户管理，不属于资产分析。
- 交易流水只展示历史，不承担主操作。
- 如果一个入口和另一个入口会进入同一类管理动作，需要合并或降级，避免重复。

### 图表约束
- 总览页只允许一个资产结构环形图。
- 规划页使用当前 vs 目标横向对比条。
- 调整方案页和大额支出模拟页使用调整前/调整后结构条。
- 不做 K 线图、复杂折线图、雷达图、Sankey、Treemap。
- 不在每个账户、每个持仓、每个卡片里放图。
- 不在同一页放多个环形图。

### 禁止倾向
- 禁止做成传统资产后台。
- 禁止一页堆很多图表。
- 禁止一页堆很多按钮。
- 禁止把账户页做成资产分析页。
- 禁止把首页做成完整管理台。
- 禁止用复杂筛选作为主体验。
- 禁止为了展示数据而增加用户不需要的指标。

## P1 规划决策

### P1-01 规划页

- [ ] 展示调整前/调整后结构条

- [ ] 支持选择用途：买车、买房、装修、教育、其他

- [ ] 展示资产结构支出前后变化
